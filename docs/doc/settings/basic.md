# 基本设置

本节介绍 Cyreneの随机点名器的所有可配置选项。所有设置均在「设置」页面中进行修改，修改后即时生效。

## 基本设置

### 语言切换

切换界面显示语言：

- **中文**：默认语言
- **English**：英文界面

切换方式：点击语言开关即可即时切换，无需重启应用。影响范围包括菜单、按钮、提示信息等界面文本。

### 深色模式

切换深色/浅色主题：

- **浅色模式**：适合白天或明亮环境使用
- **深色模式**：适合夜晚或暗光环境使用

切换方式：点击深色模式开关即可即时切换。

## 主题与显示

### 名称颜色模式

控制点名结果中名字的显示颜色：

- **渐变模式**（默认）：名字以彩虹渐变色显示，色彩丰富生动
- **自定义单色**：使用自定义颜色显示名字

选择「自定义单色」后，可分别设置浅色模式和深色模式下的名字颜色：

- **浅色模式颜色**：通过颜色选择器自定义
- **深色模式颜色**：通过颜色选择器自定义

### 界面缩放

调整界面整体大小，适应不同屏幕分辨率：

- **可选值**：75%、100%（默认）、125%、150%、175%、200%
- **实现方式**：通过 CSS 缩放实现，界面元素等比放大缩小

### 名字字体大小

调整点名结果中名字的显示大小：

- **可选值**：0.75x、1.0x（默认）、1.25x、1.5x、1.75x、2.0x
- **仅影响**：点名结果区域的名字文字大小，不影响其他界面元素

## 性能设置

以下设置用于在低性能设备上优化运行流畅度：

### 亚克力模糊

控制界面背景的毛玻璃模糊效果：

- **开启**：界面背景呈现 Windows 11 风格的亚克力模糊效果
- **关闭**：关闭模糊效果，提升低性能设备的运行流畅度

### 阴影效果

控制卡片和按钮的阴影：

- **开启**：界面元素显示阴影效果，增强层次感
- **关闭**：关闭阴影效果，减少渲染负担

### 过渡动画

控制页面切换和元素过渡动画：

- **开启**：页面切换时显示滑动过渡动画
- **关闭**：关闭过渡动画，页面直接切换，提升响应速度

## 数据管理

### 启用数据统计

控制是否记录抽取统计数据：

- **开启**（默认）：自动记录每次抽取的次数和概率分布
- **关闭**：停止记录统计数据，已有的统计数据保留但不再更新
- **公平算法开启时**：统计是算法状态的一部分，应用会自动开启并锁定此开关；关闭公平算法后才能单独关闭统计

### 数据操作密码

设置密码以保护敏感数据操作：

- **保护范围**：数据导出、数据导入、清空记录、清除全部数据
- **加密方式**：密码以 SHA-256 哈希存储，不可逆向还原
- **三种模式**：
  - **设置密码**：首次设置密码
  - **修改密码**：需先验证旧密码，再设置新密码
  - **验证密码**：执行受保护操作时需要验证

### 数据导出

将所有数据导出为 `.cyrene` 文件（JSON 格式）：

- **包含内容**：设置、名单、统计数据、抽取记录、平衡算法配置、密码哈希
- **需要密码验证**

### 数据导入

从 `.cyrene` 文件导入数据：

- **注意**：导入会覆盖当前所有数据
- **需要密码验证**
- **导入后需手动重启应用**

### 清空记录

清空所有抽取记录：

- **需要密码验证**
- **不影响**名单和统计数据

### 清除全部

清除所有数据并恢复默认设置：

- **需要密码验证**
- **不可恢复**，操作前请确认

## 平衡算法

### 启用平衡算法

控制随机抽取的公平性：

- **开启**：使用 `cyrenenameroller-balance/v3`，以绝对次数极差 2 为固定软目标，并根据历史次数动态调整下一抽概率
- **关闭**：完全随机抽取，每次概率相同

### 固定公平规则

公平参数不向用户开放修改，避免因误调或篡改导致结果失稳：

- **绝对极差 2**：目标是最高次数与最低次数之差长期稳定在 2 附近，不是百分比差
- **软目标**：所有候选者始终保留非零概率，因此允许短暂超过 2，随后通过负反馈恢复
- **多人模式**：无论允许还是禁止重复，每抽出一人都会更新临时计数并重新计算下一人概率
- **概率护栏**：限制单人当前一抽的最高概率，边界惩罚保持温和，避免近似必选或近似禁选
- **统计展示**：数据统计页面同时显示「原始概率」和「平衡概率」两列数据

### 桌面端更新

设置页检查到新版本后，Electron 和 Tauri 会由原生层下载到系统临时目录。下载完成后会校验官方发布地址、文件大小和 Windows EXE 文件头，通过校验才启动安装程序并退出当前应用。浏览器版仍通过下载页面或外部链接获取客户端。

设置页面底部展示应用的版本更新历史，数据从 `up.json` 文件加载。
