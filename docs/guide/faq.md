# 常见问题

## 安装与启动

### Q: 应用无法启动怎么办？

**Windows**:
1. 检查是否缺少 [Visual C++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe)
2. 以管理员身份运行
3. 检查杀毒软件是否误杀

**macOS**:
1. 在"系统设置" → "隐私与安全性"中允许运行
2. 右键点击应用，选择"打开"
3. 检查 Gatekeeper 设置

**Linux**:
1. 确保有执行权限：`chmod +x CyreneNameRoller.AppImage`
2. 检查是否缺少依赖库
3. 尝试用 `--no-sandbox` 参数运行

### Q: 应用启动后黑屏？

**解决方案**:
1. 检查显卡驱动是否最新
2. 在设置中禁用"模糊效果"
3. 尝试重新安装应用
4. 检查系统日志是否有错误信息

### Q: 如何更新到最新版本？

**自动更新**:
1. 应用启动时会自动检查更新
2. 有新版本时会提示
3. 点击"立即更新"即可

**手动更新**:
1. 访问 [GitHub Releases](https://github.com/Cyrene2008/CyreneNameRoller/releases)
2. 下载最新版本
3. 覆盖安装（数据会保留）

## 数据与备份

### Q: 数据存储在哪里？

**Windows**:
```
%APPDATA%/CyreneNameRoller/
```

**macOS**:
```
~/Library/Application Support/CyreneNameRoller/
```

**Linux**:
```
~/.config/CyreneNameRoller/
```

### Q: 如何备份数据？

**方法一：应用内导出**
1. 进入 **设置** 页面
2. 点击 **导出数据**
3. 选择保存位置

**方法二：手动备份**
1. 关闭应用
2. 复制数据目录
3. 保存到安全位置

### Q: 如何恢复数据？

**方法一：应用内导入**
1. 进入 **设置** 页面
2. 点击 **导入数据**
3. 选择备份文件
4. 确认导入

**方法二：手动恢复**
1. 关闭应用
2. 将备份文件夹覆盖到数据目录
3. 重新启动应用

### Q: 数据丢失了怎么办？

**自动恢复**:
应用采用双写机制，通常会自动恢复：
1. 重新启动应用
2. 检查是否自动恢复数据

**手动恢复**:
1. 检查自动备份目录（`backups/`）
2. 尝试恢复最近的备份

**如果仍无法恢复**:
1. 检查是否有手动备份
2. 查看 GitHub Issues 寻求帮助

## 功能使用

### Q: 平衡算法到底有什么用？

**A**: 平衡算法确保点名公平性：

- **问题**：传统随机点名可能让某些人连续被抽中，另一些人长期"陪跑"
- **解决**：被抽中次数越少的人，下次概率越高
- **效果**：长期来看，每个人的抽取次数更均衡

详细说明请参考 [平衡算法](/guide/balance-algorithm.md)。

### Q: 如何禁用平衡算法？

**步骤**:
1. 进入 **设置** 页面
2. 找到 **平衡算法** 部分
3. 关闭 **启用平衡算法** 开关
4. 保存设置

**适用场景**:
- 公平抽奖（需要纯随机）
- 测试功能
- 不考虑公平性的场景

### Q: 翻牌点名的历史记录会丢失吗？

**A**: 不会！

- 切换名单时，翻牌历史会保留
- 关闭应用后，历史会自动保存
- 只有手动点击"收牌"才会清空历史

### Q: 如何批量导入名单？

**方法一：文本文件**
1. 准备文本文件，每行一个名字
2. 进入 **名单管理** 页面
3. 点击 **导入**
4. 选择文件
5. 确认导入

**方法二：复制粘贴**
1. 从其他来源复制名单
2. 在名单编辑器中粘贴
3. 自动识别并分行

### Q: 白名单是什么？

**A**: 白名单是特殊人员列表：

- 白名单人员**不受平衡算法影响**
- 他们的抽取概率始终相等
- 即使被抽中多次，概率也不会变化

**适用场景**:
- 老师点名（老师不应被平衡）
- 游戏主持人（主持人不应被平衡）
- 需要保证公平性的特殊角色

## 性能问题

### Q: 点名卡顿怎么办？

**解决方案**:
1. **关闭彩虹动画**：在设置中关闭
2. **关闭粒子效果**：在性能设置中关闭
3. **关闭模糊效果**：在性能设置中关闭
4. **降低动画速度**：在性能设置中调整
5. **降低 UI 缩放**：在界面设置中调整

### Q: 应用占用内存过高？

**可能原因**:
- 点名次数过多，统计数据占用内存
- 开启了太多视觉效果

**解决方案**:
1. 定期清空抽取记录
2. 关闭不必要的视觉效果
3. 重启应用释放内存

### Q: 启动速度慢？

**解决方案**:
1. 检查是否有大量历史记录
2. 清空不需要的数据
3. 检查磁盘空间是否充足
4. 禁用自动更新检查

## 错误与异常

### Q: 提示"密码错误"？

**A**: 密码保护功能用于敏感操作：

- 如果设置了密码，需要输入密码才能：
  - 清除数据
  - 删除名单
  - 批量删除人员
- 忘记密码可以：
  - 重新设置（需要确认身份）
  - 手动删除配置文件中的密码哈希

### Q: 导入失败怎么办？

**可能原因**:
1. 文件格式不正确
2. 文件损坏
3. 版本不兼容

**解决方案**:
1. 确认文件是 `.cyrene` 或 `.json` 格式
2. 尝试用文本编辑器打开文件，检查格式
3. 导出备份文件，对比格式
4. 在 GitHub Issues 提供文件内容寻求帮助

### Q: 导出失败怎么办？

**可能原因**:
1. 磁盘空间不足
2. 没有写入权限
3. 路径不存在

**解决方案**:
1. 检查磁盘空间
2. 选择有写入权限的位置
3. 确保路径存在

## 开发与贡献

### Q: 如何从源码构建？

**步骤**:
```bash
# 克隆仓库
git clone https://github.com/Cyrene2008/CyreneNameRoller.git
cd CyreneNameRoller

# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev

# 在另一个终端启动 Electron
pnpm electron:dev

# 构建生产版本
pnpm electron:build
```
